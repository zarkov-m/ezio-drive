# AR-PAY Blog Workflow

Create practical blog plans before writing full copy.

## Workflow

1. Identify the article goal.
2. Identify the target reader.
3. Identify the offer, product, campaign, event, or update being covered.
4. Build the article structure.
5. Add CTA placement.
6. Check for clarity, search intent, and brand fit.

## Capture the minimum brief

Collect or infer these inputs:
- topic
- audience
- market/language
- business goal
- primary CTA
- supporting offer or product
- deadline or campaign date

If inputs are missing, make the smallest reasonable assumptions and label them.

## Build the article structure

Use this default structure unless the request clearly needs another shape:
- headline options
- short angle summary
- target keyword/theme
- intro hook
- 3 to 5 core sections
- CTA block
- meta description option
- social/share snippet

## Optimize for useful marketing content

Prefer:
- clear benefit-first headings
- short paragraphs
- concrete use cases
- simple CTA language
- scannable lists

Avoid:
- vague filler
- repetitive SEO wording
- fake statistics
- overlong intros

## Output formats

Depending on the request, return one of these:
- blog brief
- outline
- draft plan
- headline pack
- CTA pack
- revision notes
